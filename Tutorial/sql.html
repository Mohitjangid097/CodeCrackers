SQL Tutorial
What is SQL? Structured Query Language (SQL) is a standardised programming language for managing and manipulating relational databases. SQL allows users to create, read, update, and delete data stored in a database. It provides a simple and efficient way to communicate with databases, making it an essential skill for web developers, data analysts, and database administrators.


What is SQL
SQL is designed to be both easy to learn for beginners and powerful enough for advanced users. Some of the primary tasks you can perform with SQL include:
  • Retrieving data from a database
  • Inserting new data into a database
  • Updating existing data in a database
  • Deleting data from a database
  • Creating and managing database structures, such as tables and indexes

    • SQL has become a universal language supported by most relational database management systems (RDBMS), including MySQL, PostgreSQL, Microsoft SQL Server, Oracle, and SQLite.
• Relational Database Management Systems (RDBMS)

A Relational Database Management System (RDBMS) is a type of database management system that uses SQL to store, manage, and retrieve data in a structured format. An RDBMS organises data into tables, where each table consists of rows (records) and columns (fields). Tables can be related to one another based on common columns, known as keys, which allow you to establish relationships between data in different tables.
SQL Basics
• SQL Syntax and Keywords:

SQL syntax consists of a set of rules that determine how SQL commands and queries are structured. The main components of SQL syntax are keywords, identifiers, operators, literals, and special characters. Keywords are reserved words that have specific meanings in SQL, such as SELECT, FROM, WHERE, etc. SQL keywords are generally written in uppercase, although SQL is not case-sensitive for keywords.

A basic SQL query is structured as follows:
SELECT column1, column2, ...
FROM table_name
WHERE conditions;


In this example, SELECT and FROM are keywords. column1, column2, and table_name are identifiers, representing the names of the columns and the table in the database. The WHERE keyword is followed by conditions that filter the results.
SQL SELECT
The SELECT statement is used to query and retrieve data from one or more tables in a relational database. Here's a breakdown of the various components involved in a SELECT statement:

• SELECT Clause:
The SELECT clause specifies the columns you want to retrieve from a table. You can list the column names separated by commas, or you can use the asterisk (*) to select all columns:
SELECT* FROM table_name;


• SELECT DISTINCT
The SELECT DISTINCT clause is used to return only unique (distinct) values in the specified columns, eliminating duplicate values:
SELECTDISTINCT column1
FROM table_name;
SQL Functions
SQL functions are built-in functions that can be used to perform operations on data in tables. They are commonly used in the SELECT clause of a query, but can also be used in other parts of a query as needed. There are two main types of SQL functions: aggregate functions and scalar functions.
• Aggregate Functions (COUNT, SUM, AVG, MIN, MAX)

Aggregate functions perform calculations on a set of values and return a single value as the output. They are often used with the GROUP BY clause to perform calculations on groups of rows that share the same values in specified columns. Some common aggregate functions are:

A basic SQL query is structured as follows:
SELECT column1, column2, ...
FROM table_name
WHERE conditions;


COUNT: Returns the number of rows that match a specified condition.
SELECTCOUNT(column1)
FROM table_name;
WHERE conditions;
SQL Joins
Joins are used in SQL to combine data from two or more tables based on a related column between them. There are several types of joins that can be used to retrieve data:
• INNER JOIN:

INNER JOIN: This type of join returns rows from both tables where there is a match between the columns specified in the ON clause. If there's no match, the row will not be included in the result.
SELECT table1.column1, table2.column2
FROM table1
INNERJOIN table2
ON table1.column1 = table2.column1;


• LEFT JOIN:

LEFT JOIN (or LEFT OUTER JOIN): This join returns all the rows from the left table (table1), and the matched rows from the right table (table2). If there's no match, NULL values will be displayed for the columns from the right table.
SELECT table1.column1, table2.column2
FROM table1
LEFTJOIN table2
ON table1.column1 = table2.column1;


• RIGHT JOIN:

RIGHT JOIN (or RIGHT OUTER JOIN): This join returns all the rows from the right table (table2), and the matched rows from the left table (table1). If there's no match, NULL values will be displayed for the columns from the left table.
SELECT table1.column1, table2.column2
FROM table1
RIGHT JOIN table2
ON table1.column1 = table2.column1;


6. SQL GROUP BY and HAVING
• GROUP BY Clause:

GROUP BY Clause: Groups rows with common values in specified columns, often used with aggregate functions. It reduces the number of rows in the result set by combining rows with similar data.

SELECT category, COUNT(*) AS total_items
FROM products
GROUPBY category;


• HAVING Clause:

HAVING Clause: Filters the results of a GROUP BY query based on a condition applied to the aggregated data, similar to a WHERE clause but specifically for aggregated results.
SELECT category, COUNT(*) AS total_items
FROM products
GROUPBY category
HAVING total_items >10;
